Add three JSON-LD schema blocks to the <head> of every /rink/:slug page: LocalBusiness (as SportsActivityLocation), FAQPage, and BreadcrumbList.

These should be dynamically generated per rink from the rink's data object. The rink data should already include name, address, phone, slug, image — if it's missing any of those fields, add them.

EXAMPLE OUTPUT (for Centennial Sportsplex):

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "SportsActivityLocation",
  "name": "Centennial Sportsplex Ice Arenas",
  "image": "https://iceskatingindex.com/images/centennial-sportsplex.jpg",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "222 25th Ave N",
    "addressLocality": "Nashville",
    "addressRegion": "TN",
    "postalCode": "37203"
  },
  "telephone": "+1-615-862-8480",
  "url": "https://iceskatingindex.com/rink/centennial-sportsplex-nashville-tn",
  "sport": "Ice skating"
}
</script>

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Where do I find the public skating schedule for Centennial Sportsplex?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[content of FAQ answer]"
      }
    }
    // ... one entry per FAQ question on the page
  ]
}
</script>

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {"@type":"ListItem","position":1,"name":"Home","item":"https://iceskatingindex.com/"},
    {"@type":"ListItem","position":2,"name":"Tennessee","item":"https://iceskatingindex.com/state/tn"},
    {"@type":"ListItem","position":3,"name":"Nashville","item":"https://iceskatingindex.com/city/tn/nashville"},
    {"@type":"ListItem","position":4,"name":"Centennial Sportsplex Ice Arenas"}
  ]
}
</script>

The FAQPage schema must mirror the FAQ questions and answers actually shown on the page — so it should be generated from the same FAQ data the rink page renders, not a separate source.

Build this so every rink page automatically generates all three schema blocks from its data. After it's working on Centennial, validate one rink page at https://search.google.com/test/rich-results to confirm Google can read the schema.
